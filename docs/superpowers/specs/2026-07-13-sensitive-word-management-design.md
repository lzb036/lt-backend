# 敏感词管理与商品文本清洗设计

## 目标

在系统设置中增加仅超级管理员可见的敏感词管理页面。敏感词全系统通用，所有用户采集商品时，系统自动从商品标题和副标题中删除命中的敏感内容。

同时提供一次性服务器脚本，清理当前已经存在的待审核商品，不影响已审核、已上架或异常商品。

## 权限

- 前端菜单和路由仅超级管理员可见。
- 后端敏感词管理、模板下载和表格导入接口统一使用超级管理员权限校验。
- 普通用户不能查看或维护敏感词，但其采集商品仍应用全局敏感词规则。

## 数据结构

新增独立敏感词表，每个敏感词一条记录，字段包括：

- 唯一标识
- 敏感词内容
- 是否启用
- 创建时间
- 更新时间

敏感词内容去除首尾空白后必须非空，并设置唯一约束避免重复。

## 管理页面

“系统设置”下新增“敏感词管理”菜单，页面采用列表布局。

顶部操作：

- 下载模板
- 导入表格
- 添加敏感词汇

列表支持：

- 按敏感词搜索
- 查看敏感词、规则类型、状态和创建时间
- 新增、编辑、启用、停用和删除
- 分页

## Excel 导入

模板使用 `.xlsx` 格式，包含一列“敏感词”和示例数据。

导入规则：

- 导入内容追加到现有词库，不覆盖已有数据。
- 自动去除首尾空白。
- 空行视为无效数据并跳过。
- 文件内重复项和数据库已有项均不重复创建。
- 返回新增、重复和无效数量。

## 清洗规则

普通敏感词按原文进行文本包含匹配，命中后删除全部出现位置。

敏感词 `【】` 是特殊规则，表示删除任意 `【...】` 片段，包括左右括号和内部内容。同一字段存在多段时全部删除。

删除后进行文本整理：

- 合并连续空白。
- 清理首尾空白。
- 清理由删除操作产生的无意义空格。

若标题清洗后为空，则商品不入库，并记录明确错误。副标题允许清洗为空。

## 采集数据流

清洗在商品统一入库入口执行，使手动采集和定时采集使用相同规则。

除商品表标题外，还同步清理原始商品数据中用于标题和副标题的相关字段，包括：

- `title`
- `itemName`
- `name`
- `tagline`
- `subtitle`
- `subTitle`
- `catchCopy`
- `catchcopy`
- `catchCopyTrans`
- `saleComment`
- `sale_comment`

嵌套商品数据中存在以上字段时也应用相同清洗，防止后续编辑或上架重新读取未清洗文本。

## 初始化数据

将用户提供的敏感词作为初始全局数据写入。重复的 `即納` 只保留一条，空行和首尾空格在写入前清理。

初始化过程必须可重复执行，不得创建重复记录。

## 已有商品清理

提供一次性命令行脚本，仅处理 `review_status = "pending"` 的商品。

脚本支持：

- 默认预览，不修改数据库。
- 显式执行参数确认后才更新。
- 输出扫描数量、命中数量、修改数量和清洗后标题为空的数量。
- 同步更新商品标题和原始商品数据。
- 标题清洗为空时不删除商品，保留原值并报告，便于人工处理。

## 错误处理

- 非法文件格式返回清晰错误。
- Excel 缺少“敏感词”列时拒绝导入。
- 新增和编辑重复敏感词时返回冲突错误。
- 导入部分行无效时继续处理其他有效行，并在结果中统计。
- 敏感词读取失败时不静默修改商品文本，采集保存应报告错误。

## 测试与验证

后端测试覆盖：

- 超级管理员权限限制。
- 敏感词增删改查和唯一性。
- Excel 模板与导入统计。
- 普通敏感词删除。
- `【】` 特殊规则。
- 多词、多段和空白整理。
- 标题清洗为空。
- 原始商品数据同步清洗。
- 仅待审核商品脚本范围及预览模式。

前端验证覆盖：

- 路由和菜单仅超级管理员可见。
- API 类型及页面逻辑通过类型检查和构建检查。

按照项目约束，不额外启动前端或后端进程，不打开界面进行检查。
